# 💾 数据灾备与信源审查

## 3-2-1备份黄金法则

### 法则详解

```
3份数据副本：
  → 原始数据 + 2份备份
  → 防止单点故障

2种不同介质：
  → 如：本地硬盘 + 云存储
  → 或：NAS + 云端
  → 防止同一类型介质的集体故障

1份异地保存：
  → 备份存放在不同物理位置
  → 防本地灾害（火灾/洪水/盗窃）
```

### 具体实施方案

```
个人方案（适合大多数人）：

  主力设备（电脑/手机）
    ↓ 自动备份到
  本地NAS或外接硬盘（Time Machine）
    ↓ 定期（如每周）
  云端（iCloud/Google Drive/百度网盘）

加密要点：
  → 云端备份必须加密
  → 加密密钥不能存在同一设备上
  → 密码管理器存储加密密钥
```

## 勒索病毒防御

### 勒索病毒传播机制

```
传播路径：
  → 通常通过钓鱼邮件/恶意网站植入
  → 内网横向渗透（感染一台后扩散到局域网其他设备）
  → 加密文件后勒索（要求比特币赎金）

典型症状：
  → 文件后缀被篡改（如 .encrypted / .locked）
  → 文件无法打开
  → 桌面上出现勒索信
```

### 防御策略

```
阻止局域网横向渗透（最关键）：
  ├── 网络分区：访客网络与主网络隔离
  ├── 关闭SMBv1（445端口）
  ├── 路由器防火墙规则：禁止445端口出站
  └── NAS：不暴露到公网，不启用SMBv1

数据保护：
  ├── 3-2-1备份（即使中招也有干净副本）
  ├── 定期离线备份（拔掉外接硬盘）
  └── 云端备份（历史版本功能，防加密后同步）
```

## 搜索引擎高级语法

### 精准检索

```
site:zhihu.com "具体问题"
  → 只搜索知乎站内内容
  → 过滤低质量网站

filetype:pdf "行业报告"
  → 只搜索PDF文件
  → 适合找学术/报告类文档

"关键词"（加引号）
  → 必须包含完整短语
  → 减少无关结果

关键词1 -关键词2
  → 包含关键词1，明确排除关键词2
  → 如：iphone -评测 -评测视频

相关搜索：
  → ~关键词
  → 搜索同义词/相关词结果
```

## 假新闻交叉验证模型

### 三步验证法

```
第一步：来源核查
  ├── 这是哪个媒体/自媒体？
  ├── 该媒体有公信力吗？（查创办时间/团队/历史）
  └── 这是官方账号吗？（看认证标志）

第二步：内容核查
  ├── 事件细节有多个独立来源确认吗？
  ├── 图片/视频是否被篡改？
  │    → Google图片反向搜索：tineye.com / images.google.com
  ├── 数据/统计是否断章取义？
  │    → 查原文，查计算基数
  └── 逻辑是否明显漏洞？

第三步：时间线核查
  ├── 这件事是什么时候发生的？
  ├── 是否是旧闻重新包装？
  │    → 去搜索事件发生时的新闻
  └── 事件后续发展如何？（很多假新闻只截取片段）
```
