# Dataset (数据集)

## 一句话理解

模型是数据的"压缩版"——训练数据决定模型知道什么、不知道什么、相信什么。数据质量直接决定了模型能力的上限，数据是AI大模型真正的"精神食粮"。

## 核心必背

> **核心必背**：大模型的性能上限由数据质量决定，而非模型架构。GPT-3训练了300B tokens，Chinchilla发现同等算力下用更多数据（1.4T tokens）训练更小的模型效果更好。数据质量（多样性、去噪程度、有害内容过滤）比单纯的数量更重要——Garbage in, Garbage out。

## 数据质量的四个维度

### 1. 质量（Quality）
- 去噪：去除HTML标签、模板文本、重复内容
- 去毒：过滤有害、偏见内容
- 标注质量：问答对、偏好数据的标注准确性

### 2. 多样性（Diversity）
- 领域覆盖：学术、代码、对话、文学等
- 语言覆盖：多语言数据的比例
- 观点覆盖：避免单一视角的过度放大

### 3. 时效性（Recency）
- 知识截断：ChatGPT-4知识截止2023年4月
- 持续预训练：用新数据更新模型知识
- 动态数据策略：根据任务需求动态采样

### 4. 规模（Scale）
- Token数量：通常指B（10亿）为单位
- 质量-数量权衡：Common Crawl占约60%原始数据，但经清洗后仅剩约4%

## 主流训练数据集

| 数据集 | 规模 | 特点 |
|--------|------|------|
| Common Crawl | ~7T tokens | 互联网最大语料，需深度清洗 |
| The Pile | ~825GB | 22个高质量子数据集的混合 |
| Red Pajama | 1.2T tokens | 从Common Crawl提取的高质量子集 |
| WebText | ~40GB | Reddit高赞外部链接 |
| GPT-4o数据 | 未知 | OpenAI未公开 |

## 数据工程的关键性

Meta的LLaMA系列证明：在更小但更高质量的1.4T tokens上训练的65B模型，可以超越GPT-3的175B模型。这说明**数据质量>>数据数量**。这个发现改变了整个行业的训练范式。

## 数据偏见（Data Bias）

数据是模型的"世界观"。如果数据中男性医生多、女性护士多，模型会学会这种刻板印象。更危险的是，模型会自信地强化这些偏见，因为这是它从数据中"学到"的客观规律。

## 合成数据（Synthetic Data）

近年重要趋势：用强模型生成训练数据，如：
- GPT-4生成问答数据
- AlphaCode生成代码数据
- 关键挑战：防止模型在自己生成的数据上训练导致"能力崩溃"（Model Collapse）
