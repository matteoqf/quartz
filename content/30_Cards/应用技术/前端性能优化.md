# 前端性能优化

## 概述

前端性能优化是通过技术手段提升页面加载速度、交互响应能力和用户体验的系统性方法。优化目标通常围绕Core Web Vitals三大指标展开：LCP（最大内容绘制）、FID（首次输入延迟）和CLS（累积布局偏移）。

## 一、加载性能优化

### 1.1 资源加载优化

**减少HTTP请求**
- 合并CSS/JS文件，减少请求数量
- 使用CSS Sprites合并小图标
- 图片懒加载，非首屏图片延迟加载
- 接口合并与批量请求

**资源压缩与压缩传输**
```
# 构建配置示例
- terser 压缩JavaScript
- cssnano 压缩CSS
- image-webpack-loader 压缩图片
- 启用Gzip/Brotli压缩
```

**使用CDN加速**
- 静态资源部署到CDN
- 就近访问，减少网络延迟
- 提供边缘缓存能力

### 1.2 渲染阻塞优化

**CSS优化**
- 将关键CSS内联到HTML中
- 非关键CSS使用`media`属性或`loading="lazy"`
- 避免使用`@import`引入CSS

**JavaScript优化**
```
<!-- 现代浏览器支持 -->
<script defer src="app.js"></script>
<!-- defer：HTML解析完成后执行 -->
<!-- async：下载完成即执行，不保证顺序 -->
```

- 减少第三方脚本影响
- 使用`Intersection Observer`实现懒加载
- 避免同步脚本阻塞渲染

### 1.3 缓存策略

| 资源类型 | 缓存策略 | 实现方式 |
|---------|---------|---------|
| HTML | 强缓存+协商缓存 | Cache-Control: no-cache |
| 静态资源 | 长期强缓存 | hash命名 + max-age=31536000 |
| API数据 | 协商缓存 | Last-Modified / ETag |
| 用户数据 | 内存缓存 | sessionStorage/localStorage |

## 二、渲染性能优化

### 2.1 关键渲染路径优化

**减少DOM复杂度**
- 保持DOM结构扁平化
- 避免深层嵌套
- 减少DOM节点数量

**优化CSS选择器**
- 避免过深的选择器嵌套
- 避免使用通配符选择器
- 使用类选择器替代复杂选择器

### 2.2 重排与重绘优化

**触发重排的操作**
- 改变元素尺寸（width, height, padding, margin）
- 改变元素位置（top, left）
- 添加/删除元素
- 获取布局信息（offsetWidth, scrollTop）

**优化技巧**
```javascript
// 批量DOM操作
const fragment = document.createDocumentFragment();
// 多次DOM操作
fragment.appendChild(/* ... */);
container.appendChild(fragment);

// 使用CSS transform替代top/left
element.style.transform = 'translateX(100px)'; // 不触发重排
element.style.top = '100px'; // 触发重排

// 读写分离，避免交错读写触发重排
// 坏例子：
element.style.width = element.offsetWidth + 10 + 'px'; // 读+写+读

// 好例子：
const width = element.offsetWidth; // 读
element.style.width = width + 10 + 'px'; // 写
```

### 2.3 动画性能优化

**使用GPU加速**
```css
/* 开启GPU加速 */
.animated-element {
  will-change: transform;
  transform: translateZ(0);
  backface-visibility: hidden;
}
```

**动画属性选择**
| 属性 | 性能 | 说明 |
|-----|-----|-----|
| transform | 优 | GPU加速，仅影响组合属性 |
| opacity | 优 | GPU加速，不触发重排重绘 |
| filter | 中 | 部分GPU加速 |
| width/height | 差 | 触发重排 |
| background-color | 差 | 触发重绘 |

## 三、执行效率优化

### 3.1 JavaScript执行优化

**事件优化**
```javascript
// 事件委托 - 减少事件处理器数量
document.querySelector('ul').addEventListener('click', (e) => {
  if (e.target.matches('li')) {
    handleItemClick(e.target);
  }
});

// 防抖与节流
function debounce(fn, delay) {
  let timer = null;
  return function(...args) {
    clearTimeout(timer);
    timer = setTimeout(() => {
      fn.apply(this, args);
    }, delay);
  };
}

function throttle(fn, limit) {
  let inThrottle = false;
  return function(...args) {
    if (!inThrottle) {
      fn.apply(this, args);
      inThrottle = true;
      setTimeout(() => inThrottle = false, limit);
    }
  };
}
```

**避免内存泄漏**
- 及时清理定时器（setTimeout/setInterval）
- 移除事件监听器
- 避免闭包导致的内存无法释放
- 使用WeakMap/WeakSet存储对象引用

### 3.2 长任务拆分

```javascript
// 使用requestIdleCallback拆分长任务
function processInChunks(items, chunkSize = 100) {
  let index = 0;
  
  function processChunk(deadline) {
    while (index < items.length && deadline.timeRemaining() > 0) {
      processItem(items[index]);
      index++;
    }
    
    if (index < items.length) {
      requestIdleCallback(processChunk);
    }
  }
  
  requestIdleCallback(processChunk);
}
```

## 四、感知性能优化

### 4.1 加载状态优化

**骨架屏**
```jsx
function Skeleton({ width, height }) {
  return (
    <div 
      className="skeleton"
      style={{ width, height }}
    />
  );

// 使用
{isLoading ? (

) : (
  <Content />
)}
```

**渐进式加载**
- 先展示文字内容
- 图片逐张加载
- 功能模块按需加载

### 4.2 交互反馈优化

- 按钮点击提供即时反馈（:active态）
- 使用loading状态指示操作进度
- 缓存操作结果提供乐观更新
- 失败时提供明确错误提示

## 五、监控与度量

### 5.1 性能指标

| 指标 | 含义 | 优化目标 |
|-----|------|---------|
| FCP | 首次内容绘制 | < 1.8s |
| LCP | 最大内容绘制 | < 2.5s |
| FID | 首次输入延迟 | < 100ms |
| CLS | 累积布局偏移 | < 0.1 |
| TTFB | 首字节时间 | < 800ms |

### 5.2 性能监控方案

```javascript
// 使用Performance API
const observer = new PerformanceObserver((list) => {
  for (const entry of list.getEntries()) {
    console.log(entry.name, entry.startTime, entry.duration);
  }
});

observer.observe({ entryTypes: ['paint', 'longtask', 'layout-shift'] });

// 使用Web Vitals库
import { getCLS, getFID, getLCP } from 'web-vitals';

getCLS(console.log);
getFID(console.log);
getLCP(console.log);
```

## 六、框架特定优化

### 6.1 React优化

**减少不必要的渲染**
```jsx
// 使用React.memo缓存组件
const MemoizedComponent = React.memo(ExpensiveComponent);

// 使用useMemo缓存计算结果
const memoizedValue = useMemo(() => computeExpensiveValue(a, b), [a, b]);

// 使用useCallback缓存回调
const handleClick = useCallback(() => {
  doSomething(a, b);
}, [a, b]);
```

**列表渲染优化**
```jsx
function ItemList({ items }) {
  return (
    <WindowedList
      height={400}
      itemCount={items.length}
      itemSize={50}
    >
      {({ index, style }) => (
        <div style={style}>{items[index].name}</div>
      )}
    </WindowedList>
  );
}
```
