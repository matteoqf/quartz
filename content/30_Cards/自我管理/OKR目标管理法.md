# OKR目标管理法

## 概述

OKR（Objectives and Key Results，目标与关键结果）是由Intel创始人安迪·格鲁夫提出、后被Google等公司广泛采用的目标管理框架。其核心是通过设定具有挑战性的目标（O）和可量化的关键结果（KR），实现组织的聚焦、对齐与追踪。

## 核心结构

### Objective（目标）
- 描述性、定性的方向指引
- 回答"我们要去哪里"
- 应具有激励性、鼓舞人心
- 通常以动词开头（如：成为...、实现...、打造...）

### Key Results（关键结果）
- 量化性、可度量的指标
- 回答"如何衡量我们是否到达"
- 通常为2-5条
- 符合SMART原则（具体、可衡量、可实现、相关、有时限）

## 设定流程

1. **明确愿景** → 思考长期目标和方向
2. **设定Objective** → 凝练1-3个核心目标
3. **拆解Key Results** → 为每个O设定2-5个KR
4. **自评分级** → 根据完成程度打分（0-1.0）
5. **复盘与迭代** → 周期结束后总结经验，调整下一周期OKR

## 评分标准

| 分数 | 含义 |
|------|------|
| 1.0 | 完全达成（理想状态，通常难以达到） |
| 0.7 | 大部分达成（良好表现） |
| 0.3-0.6 | 部分达成（需分析差距原因） |
| 0-0.3 | 未达成（需要重大改进） |

> 注：OKR鼓励设定有挑战性的目标，得分0.6-0.7通常被认为是优秀表现。

## OKR与KPI的区别

| 维度 | OKR | KPI |
|------|-----|-----|
| 导向 | 目标导向、创新驱动 | 指标导向、绩效挂钩 |
| 透明度 | 全员公开 | 通常局部可见 |
| 评分 | 不直接关联薪酬 | 通常与考核强关联 |
| 目标设定 | 挑战性目标（70%完成即优秀） | 现实可达目标（100%为标准） |

## 实践要点

- **聚焦**：一个周期内关键目标不超过3-5个
- **对齐**：上下级OKR需保持一致性与关联性
- **跟踪**：定期检查进度，避免期末才发现偏差
- **复盘**：周期结束时进行反思与经验沉淀
- **氛围**：营造敢于设定挑战性目标的文化

## 常见周期

- 公司/团队：季度OKR（3个月）
- 个人：月度OKR或季度OKR
- 长期愿景：年度战略OKR
