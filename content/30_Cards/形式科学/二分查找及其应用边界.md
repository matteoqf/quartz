# 二分查找及其应用边界

## 核心思想

二分查找（Binary Search）通过每次将搜索区间减半来定位目标元素。其本质是利用**有序性**将 $O(n)$ 的线性搜索降为 $O(\log n)$ 的对数级搜索。

---

## 算法模板

```
left, right = 0, len(array) - 1
while left <= right:
    mid = left + (right - left) // 2
    if array[mid] == target:
        return mid
    elif array[mid] < target:
        left = mid + 1
    else:
        right = mid - 1
return -1
```

关键点：

- `left + (right - left) // 2` 防止整数溢出
- 循环条件是 `left <= right`（区间非空）
- `mid + 1` / `mid - 1` 避免死循环

---

## 适用条件

1. **有序数组**（升序/降序）
2. **随机访问**（数组支持 O(1) 下标访问）
3. **单调性判断**：满足 `f(mid) <= f(mid+1)` 的单调函数求极值

---

## 常见应用模式

### 1. 精确查找

数组中目标值存在，返回下标；不存在返回 -1。

### 2. 下界查找（左边界）

返回第一个 `>= target` 的位置。

```
while left < right:
    mid = left + (right - left) // 2
    if array[mid] < target:
        left = mid + 1
    else:
        right = mid
```

### 3. 上界查找（右边界）

返回最后一个 `<= target` 的位置。

```
while left < right:
    mid = left + (right - left + 1) // 2  # 向上取整
    if array[mid] > target:
        right = mid - 1
    else:
        left = mid
```

### 4. 单调函数求极值

如 `f(x)` 单调，先增后减（或反之），二分求峰值/谷值。

---

## 应用边界与失效场景

### 边界一：数据结构的根本限制

- **链表**：不支持 O(1) 随机访问，二分无法直接应用。若强行二分需借助跳表等扩展结构。
- **哈希表**：哈希表按 key 直接寻址，时间复杂度 O(1)，二分不适用。

### 边界二：有序性被破坏

- **无序数组求最值**：只能线性扫描，二分无效。
- **旋转数组查找**（e.g., `[4,5,6,7,0,1,2]` 找 target）：需要改进的二分，但需满足「两段有序」的前提。

### 边界三：近似 vs 精确

- **近似查找**（最近邻、K 近邻）：二分只能定位精确点，无法处理模糊匹配。
- **浮点精度问题**：二分结束条件需设置 `eps`，否则可能无限循环。

### 边界四：答案空间离散且无显式数组

- 「在 `[L, R]` 范围内求满足条件 P 的最小值」类型问题，需**二分答案**（隐式二分）。但若答案空间无序（如组合优化），二分失效。

### 边界五：多维搜索

- 二分只适用于一维有序。二维矩阵查找（如 Search a 2D Matrix）需按「行首尾 + 列首尾」做两次二分或从右上角 DFS。

---

## 复杂度

| 维度 | 复杂度                              |
| ---- | ----------------------------------- |
| 时间 | $O(\log n)$                         |
| 空间 | $O(1)$（迭代）；$O(\log n)$（递归） |

---

## 本质

二分的本质是**利用单调性在解空间上做「猜-验证」的快速剪枝**。只要解空间具备可比较的单调关系（不一定是数组），二分思想即可迁移——这就是「二分答案」的理论基础。
