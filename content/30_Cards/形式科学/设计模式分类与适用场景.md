# 设计模式分类与适用场景

## 概述

设计模式（Design Pattern）是对软件设计中普遍存在问题的**通用可重用解决方案**。由四人帮（GoF）在《设计模式：可复用面向对象软件的基础》中系统化提出，分为三大类共23种模式。

---

## 一、创建型模式（Creational Patterns）

解决对象**实例化**的封装问题，将对象创建与使用解耦。

| 模式 | 核心思想 | 适用场景 |
|------|----------|----------|
| **单例模式** | 确保一个类仅有一个实例 | 配置管理、数据库连接池、日志系统 |
| **工厂方法** | 定义创建对象的接口，让子类决定实例化哪个类 | 数据库驱动切换、插件系统 |
| **抽象工厂** | 创建一系列相关对象而无须指定具体类 | 跨平台UI组件、多数据库支持 |
| **建造者模式** | 分步构建复杂对象 | SQL查询构建、文档格式转换 |
| **原型模式** | 通过克隆已有对象创建新对象 | 对象复制、缓存快照 |

---

## 二、结构型模式（Structural Patterns）

关注对象和类的**组合**，形成更大的结构。

| 模式 | 核心思想 | 适用场景 |
|------|----------|----------|
| **适配器模式** | 将不兼容接口转换为可兼容接口 | 第三方库集成、遗留系统对接 |
| **桥接模式** | 分离抽象部分与实现部分 | 跨平台图形引擎、设备驱动 |
| **装饰器模式** | 动态添加职责 | 日志增强、缓存层、权限校验 |
| **组合模式** | 树形结构统一处理个体与整体 | 文件系统、UI组件树、组织架构 |
| **外观模式** | 提供统一高层接口 | 复杂子系统封装、API网关 |
| **享元模式** | 共享细粒度对象 | 字符串池、图标缓存、棋牌游戏 |
| **代理模式** | 控制对原对象的访问 | 延迟加载、安全控制、远程调用 |

---

## 三、行为型模式（Behavioral Patterns）

关注对象之间的**职责分配**与通信协作。

| 模式 | 核心思想 | 适用场景 |
|------|----------|----------|
| **策略模式** | 定义一系列算法并可互换 | 支付方式、排序算法、促销计算 |
| **观察者模式** | 一对多依赖，状态变化通知 | 事件系统、消息订阅、数据绑定 |
| **命令模式** | 请求封装为对象 | 撤销/重做、任务队列、宏录制 |
| **模板方法** | 定义算法骨架，子类实现特定步骤 | 框架钩子、数据处理流程 |
| **迭代器模式** | 顺序访问集合元素 | 集合遍历、数据库游标 |
| **职责链模式** | 沿链传递请求直到被处理 | 审批流程、中间件链、日志级别 |
| **状态模式** | 对象内部状态改变时改变行为 | 状态机、订单流程、游戏状态 |
| **访问者模式** | 数据结构与操作分离 | 编译器AST处理、报表生成 |
| **中介者模式** | 对象间通信通过中介协调 | UI组件通信、多模块协调 |
| **解释器模式** | 定义语法并解释 | 配置文件解析、正则表达式 |
| **备忘录模式** | 捕获对象状态用于恢复 | 撤销功能、游戏存档 |

---

## 模式选择决策图

```
需要对象创建?
│ 是
├─ 仅需一个实例? → 单例模式
├─ 创建逻辑需解耦? → 工厂方法
├─ 创建产品族? → 抽象工厂
├─ 构建步骤多/可变? → 建造者模式
└─ 已有实例需复制? → 原型模式
│ 否
├─ 需要结构组合?
│  ├─ 接口不兼容? → 适配器模式
│  ├─ 抽象与实现分离? → 桥接模式
│  ├─ 动态添加职责? → 装饰器模式
│  ├─ 整体与部分树形结构? → 组合模式
│  ├─ 提供统一入口? → 外观模式
│  ├─ 细粒度对象共享? → 享元模式
│  └─ 间接访问控制? → 代理模式
└─ 需要对象协作/通信?
   ├─ 算法可互换? → 策略模式
   ├─ 状态变化通知? → 观察者模式
   ├─ 请求封装/队列? → 命令模式
   ├─ 算法骨架固定? → 模板方法
   ├─ 集合遍历? → 迭代器模式
   ├─ 请求链式处理? → 职责链模式
   ├─ 状态影响行为? → 状态模式
   ├─ 操作与数据分离? → 访问者模式
   ├─ 对象间解耦通信? → 中介者模式
   ├─ 语法解释执行? → 解释器模式
   └─ 状态快照恢复? → 备忘录模式
```

---

## 补充：反模式（Anti-Patterns）

避免使用的常见错误模式：

- **上帝类（God Object）**：一个类承担过多职责
- **魔法数字（Magic Numbers）**：硬编码的未命名常量
- **回调地狱（Callback Hell）**：深层嵌套回调
- **复制粘贴编程**：代码重复未抽象
