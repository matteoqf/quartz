# 💡 Context Window上下文窗口

> **本质**：LLM在单次推理中能处理的最大token数量，决定了可以同时考虑的信息范围。

---

## 🔬 发现历程
GPT-2(2019)上下文1024 token → GPT-3(2020) 2048 token → GPT-4(2023) 128K token → Claude/Gemini 1M token。上下文窗口的扩展经历了5年增长超过100倍。

## 📊 关键参数
| 参数 | 数值/标准 |
|------|-----------|
| 典型范围 | 4K（GPT-3.5）到 128K（GPT-4 Turbo） |
| 最新突破 | 1M token（Gemini 1.5, Claude 3） |
| Memory长度 | 上下文内有效信息通常只有前30-50% |

## ⚠️ 误区排雷
- ❌ 误区一：上下文窗口越大越好 → 正确的是：超出窗口后召回率急剧下降，长上下文需要稀疏注意力等优化
- ❌ 误区二：模型能平等利用窗口内所有信息 → 正确的是：存在"中间丢失"问题，中间部分信息权重低于首尾

## 🎯 行动要点
- 信息放置：将最重要结论放在窗口开头或结尾（位置编码的非线性效应）
- 长文档处理：使用RAG或摘要压缩，而非全部塞入上下文
- 注意力优化：Streaming LLM、LongLoRA等技术在保持核心注意力的同时延长上下文
