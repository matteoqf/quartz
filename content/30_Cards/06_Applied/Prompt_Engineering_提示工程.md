# Prompt Engineering (提示工程)

## 定义

Prompt Engineering（提示工程）是指研究和实践如何优化提示词，以获得更准确、更可靠、更有用的模型输出的系统性方法论。它不是简单的"问问题"，而是一门将输入文本视为编程语言来精心设计的工程学科。

## 核心必背

> **核心必背**：提示工程是连接人类意图与模型能力的桥梁，优秀的提示工程能使同等模型产生质的飞跃。

## 核心技巧

### 1. 明确性原则
指令必须清晰无歧义。"写一首关于秋天的诗"太宽泛；"写一首五言绝句，主题是秋天的落叶，押韵平水韵"则具体得多。

### 2. 结构化输出
当需要特定格式时，在 Prompt 中明确指定。例如："请以JSON格式返回，包含title、content、tags三个字段"。

### 3. 分解复杂任务
将大型任务拆解为链状子任务，每个步骤独立优化。复杂问题一次问不如分步问效果稳定。

### 4. 利用示例（Few-shot Learning）
通过提供2-3个输入-输出示例，让模型学习期望的模式，适用于有特定格式或逻辑要求的场景。

### 5. 思维链引导（CoT）
对于推理类问题，加入"让我们一步步思考"之类的引导语，激发模型的链式推理能力。

## 迭代优化方法

创建 Prompt 后通过多轮测试评估效果，记录每次修改带来的差异。建立 Prompt 版本库，保留有效变体。关注：输出准确性、格式一致性、边界情况处理。

## 工具与框架

LangChain、Semantic Kernel等框架提供了 Prompt 模板化和链式组合能力。Dspy等框架尝试用程序化方式自动优化 Prompt 链。生产环境中通常需要对 Prompt 进行版本管理和灰度发布。

## 工程化建议

企业应建立内部的 Prompt 最佳实践库，对关键场景的提示词进行标准化。Prompt 质量应纳入QA流程，尤其在客服、诊断、内容生成等高风险场景。提示工程不是一次性工作，而是随模型版本迭代持续优化的过程。
