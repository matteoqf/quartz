# 🔐 密码学与数据安全

## 撞库攻击原理

### 攻击机制

```
撞库（Credential Stuffing）：
  → 用户在A网站注册了账号（邮箱+密码）
  → A网站被黑客拖库（用户数据泄露）
  → 黑客用同一套账号密码组合
    去B网站/C网站/D网站...批量尝试登录

本质：用户习惯在多个网站用相同密码
     → 一个网站泄露 = 所有网站沦陷
```

### 防御策略：密码分级

| 等级 | 账户类型 | 密码策略 | 管理工具 |
|------|---------|---------|---------|
| 🔴 最高级 | 金融/支付/邮箱主账号 | 16位随机，大小写+数字+符号 | 物理密钥（YubiKey）|
| 🟠 高级 | 社交媒体/云存储 | 12位随机，密码管理器生成 | 密码管理器 |
| 🟡 中级 | 一般网站/论坛 | 12位，不同密码 | 密码管理器 |
| 🟢 低级 | 一次性/边缘网站 | 通用密码（泄露影响小） | 记住即可 |

### 密码生成规则

```
随机密码生成公式（不依赖工具）：
  ① 选一句你记得住的中文句子
  ② 取每字拼音首字母
  ③ 数字替换（如 a→4, o→0, i→1）
  ④ 大小写混合
  ⑤ 加入特殊符号分隔

示例：
  "我的生日是1995年10月1日"
  → WDshrS1995n10y1r
  → WdshrS1995N10y1R!@
```

## 两步验证（2FA）与TOTP

### 2FA类型安全性排序

```
最安全 → 最不安全：

1. 硬件安全密钥（YubiKey/Google Titan）
   → 公私钥对，无法钓鱼，无法钓鱼
   → 物理丢失可补购

2. TOTP验证器（Google Authenticator/Authy）
   → 30秒动态码，离线可用
   → 绑定设备，手机丢失 = 需要备份码

3. 短信验证码（SMS）
   → SIM卡被劫持（SIM swapping）可被拦截
   → 社会工程学攻击风险高

4. 邮件验证码
   → 邮箱被盗 = 全线沦陷
   → 比短信好，但不如TOTP
```

### TOTP工作原理

```
TOTP = Time-based One-Time Password

算法：
  共享密钥（注册时绑定）+ 当前Unix时间戳（30秒窗口）
    → 通过SHA-1哈希 → 6位数字动态码

安全性：
  → 每次密码不同，无法钓鱼
  → 离线可用（不依赖网络）
  → 设备丢失 = 需要备份码或账户恢复
```

### OAuth授权登录风险

```
常见授权场景：
  "用微信登录" / "用Google登录" / "用Apple登录"

OAuth授权内容（你授权给第三方）：
  ├── 基础信息（昵称/头像/邮箱）
  ├── 可能包含：好友列表/发帖记录
  └── ❌ 不会获取你的账号密码

风险点：
  → 第三方网站信誉参差不齐
  → 授权范围可能超出必要（要求"获取好友列表"）
  → 建议：用主邮箱的随机密码注册 > OAuth授权
```

## 数据备份3-2-1法则

```
3份数据副本
  → 原始文件 + 2份备份

2种不同介质
  → 如：本地硬盘 + 云端
  → 或：SSD + 机械硬盘

1份异地保存
  → 如：云端备份（在异地/不同机房）
  → 防本地灾难（火灾/盗窃/水灾）
```
